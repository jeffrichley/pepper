# Pepper Init вЂ” Runtime/Dev Separation Design

**Date:** 2026-04-06
**Status:** Approved

## Problem

Pepper's repo serves double duty as both source code and runtime workspace. When Claude Code spawns inside the repo, it loads the full assistant persona (hooks, MCP servers, skills, identity) вЂ” making development awkward. Developers get the Pepper personality instead of a clean coding environment.

## Solution

Separate the runtime workspace from the source code repo:

- **`~/.pepper/`** вЂ” runtime workspace where Claude Code runs as Pepper the assistant
- **`pepper/` repo** вЂ” source code only, clean dev environment
- **`pepper init`** вЂ” CLI command that generates the runtime workspace from templates in the installed package

## Architecture

| | Dev (pepper repo) | Runtime (~/.pepper/) |
|---|---|---|
| Purpose | Write code, run tests | Run pepper as assistant |
| `.claude/` | Minimal dev config or none | Full config with hooks, MCP, skills |
| Vault | Not present | `~/.pepper/Memory/` |
| Claude Code persona | Generic developer | Pepper |

## Entry Points (pyproject.toml)

```toml

pepper = "pepper.cli:app"
pepper-channel = "pepper.channel.server:main"
pepper-discord = "pepper.integrations.discord.mcp_server:main"
```

## Hybrid Referencing (Code vs Config)

Static config files (CLAUDE.md, .mcp.json, settings.json) are generated once by `pepper init`. Hooks, skills, and MCP servers are referenced from the installed package so they stay current with `uv sync`.

### settings.json (generated)

```json
{
  "hooks": {
    "SessionStart": [{
      "command": "python -m pepper.hooks.session_start_context",
      "timeout": 10000
    }],
    "PreCompact": [{
      "command": "python -m pepper.hooks.pre_compact_flush",
      "timeout": 15000
    }],
    "SessionEnd": [{
      "command": "python -m pepper.hooks.session_end_flush",
      "timeout": 15000
    }]
  }
}
```

### .mcp.json (generated)

```json
{
  "mcpServers": {
    "pepper-channel": { "command": "pepper-channel" },
    "pepper-discord": { "command": "pepper-discord" }
  }
}
```

Updating the repo + `uv sync` updates runtime behavior without re-running `pepper init`.

### Vault path discovery

Hooks and MCP servers need to find the vault. Since Claude Code runs from `~/.pepper/`, relative paths (`./Memory/`) work by default. For cases where code runs outside that directory, `config.toml` stores the absolute vault path and `pepper.hooks.shared` reads it.

**Fallback:** If the hybrid approach proves problematic (e.g., Claude Code can't resolve installed entry points for hooks), fall back to generating all files via `pepper init`, requiring re-run on updates.

## Pepper Channel вЂ” Python Rewrite

The existing TypeScript/Bun pepper-channel MCP server is rewritten in Python using the `mcp` Python SDK. This eliminates the Bun/Node dependency, giving a single-language stack (Python + uv).

The channel server is a message router:
- Maintains a routing table: `chat_id -> { source, timestamp }`
- Routes messages between integrations (Discord, etc.) and Claude Code
- Configurable PORT and TTL

## CLI Commands

```
pepper init [--migrate]    # Generate ~/.pepper/ runtime workspace
                           # --migrate moves existing Memory/ from repo
pepper start               # Start MCP servers and integrations
pepper stop                # Stop running services
```

## Migration from Current State

### `pepper init` (fresh)
Creates `~/.pepper/` with empty scaffold vault files (IDENTITY.md, SOUL.md, etc.).

### `pepper init --migrate`
Moves existing `pepper/Memory/` contents into `~/.pepper/Memory/` and updates paths.

### Repo cleanup after migration
- Remove `Memory/`, `.claude/`, `.mcp.json`, `CLAUDE.md` from repo root
- Move existing hooks under `src/pepper/hooks/`
- Move existing skills under `src/pepper/skills/`
- Update `.gitignore` to exclude runtime artifacts
- Optionally add a minimal `.claude/` with dev-only settings

## What Stays in the Repo

- `src/pepper/` вЂ” all source code (CLI, hooks, skills, channel, integrations)
- `tests/` вЂ” test suite
- `pyproject.toml`, `uv.lock`
- `scripts/` вЂ” start/stop scripts (updated for new paths)
- `docs/`
- `.gitignore` updated

## Dev Workflow

1. `cd pepper/` вЂ” Claude Code spawns clean, no assistant persona
2. Write code, run tests normally
3. `uv sync` to reinstall the package after changes
4. Test runtime: `cd ~/.pepper && claude`

## Testing

- **Hooks:** Unit tests with mocked vault path (existing tests adapted)
- **`pepper init`:** Tests using temp directory instead of `~/.pepper/`
- **Channel MCP server:** Tests ported from existing TypeScript coverage
- **Integration test:** `pepper init` into temp dir, verify all files generated correctly
